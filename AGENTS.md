# Repository Guidelines

## Project Structure & Module Organization
This is a TypeScript monorepo using npm workspaces.
- `client/` — Vite + React UI, Tailwind CSS. Source in `client/src/` (`components/`, `contexts/`, `hooks/`, `pages/`).
- `server/` — Express + Socket.IO backend. Source in `server/src/` (`game/`, `socket/`, `utils/`).
- `shared/` — Shared TypeScript types/utilities consumed by client and server. Source in `shared/src/`.
- Build outputs go to each package’s `dist/` (generated by `tsc`/Vite).

## Build, Test, and Development Commands
Run from repo root:
- `npm run dev` — Start client and server concurrently (Vite + `tsx watch`).
- `npm run dev:client` — Client dev server only.
- `npm run dev:server` — Server dev process only.
- `npm run build` — Build shared, then client, then server.
- `npm run start` — Run the built server (`server/dist/index.js`).

Package-level examples:
- `npm run build --workspace=client`
- `npm run dev --workspace=shared`

## Coding Style & Naming Conventions
- Language: TypeScript with ES modules (`type: "module"`).
- Keep formatting consistent with existing files (no formatter configured).
- React components: `PascalCase` filenames and component names (e.g., `GameBoard.tsx`).
- Hooks: `useX` naming (e.g., `useSocket.ts`).
- Contexts/providers: `XContext` and `XProvider`.

## Testing Guidelines
No test framework is currently configured. If you add tests, include a tool choice (e.g., Vitest/Jest) and wire a `test` script at root and package level.

## Commit & Pull Request Guidelines
Git history currently shows a single commit message: `baseline`. No strict convention is enforced yet.
- Use short, imperative summaries (e.g., “Add lobby view”, “Fix socket reconnection”).
- PRs should include a concise description, linked issues if applicable, and UI screenshots for client-facing changes.

## Configuration & Environment
- The server runs via `server/src/index.ts` in dev and `server/dist/index.js` in prod.
- Shared types are built from `shared/src/` and imported as `@uno-web/shared`.
