# Task Plan

## 目标
- 删除项目中冗余部分
- 优化整体架构
- 补齐并规范化全部测试
- 补全前后端缺失功能

## 项目范围（基于当前仓库）
- 前端：`client/`（Vite + React + Tailwind）
- 后端：`server/`（Express + Socket.IO）
- 共享：`shared/`（类型与公共工具）

## 任务拆解与顺序
1. 现状盘点与基线确认：梳理目录结构、依赖与脚本；确认关键业务流（创建房间、加入房间、准备、开始游戏、出牌、抽牌、结束回合、挑战）。输出“当前功能覆盖矩阵”与“缺失项清单”。
2. 冗余清理清单：识别并标注未使用的代码/资源/依赖（例如未使用的导入、重复实现、空目录、无引用工具函数）；输出“待删除清单”并准备删除顺序。
3. 架构优化方案：明确模块职责边界（client 只处理 UI 与交互、server 只处理规则与状态、shared 只包含纯类型与无状态工具）；确认共享逻辑是否需要下沉到 `shared/`（例如纯函数工具），并列出需要移动/合并的文件与替换路径。
4. 冗余删除与重构落地：按清单逐项删除冗余代码与依赖；合并重复实现（如状态转换/映射逻辑）；清理未使用的事件、类型或变量；更新 import 与引用关系，确保 `build` 可通过。
5. 统一事件与状态流：校验 Socket 事件命名与负载定义（`shared/src/types/events.ts`）与前后端实现一致；补齐缺失的事件处理或错误处理（连接断开、玩家离开、房间被销毁、游戏结束等），形成完整状态机流转。
6. 前端缺失功能补齐：补上游戏流程缺口与 UI 交互（例如玩家加入/离开提示、游戏结束后返回大厅/再开局、错误提示可视化、断线重连提示）；与现有 `GameContext`/`pages` 对齐。
7. 后端缺失功能补齐：补齐游戏生命周期处理（房间解散、游戏结束后状态回收、玩家掉线的回合处理与规则补偿）；补齐挑战/惩罚与出牌边界条件；确保 `GameManager` 状态不会出现非法转移。
8. 测试框架选型与规范化：选择统一测试框架（建议 Vitest + @testing-library/react + jsdom + supertest/socket.io-client）；在根与各 workspace 增加 `test` 脚本；建立测试目录与命名规范（例如 `*.test.ts`/`*.test.tsx`），并在 `shared/` 提供纯函数单测模板。
9. 共享层测试补齐：覆盖 `shared/` 纯函数与类型辅助（例如卡牌判定、玩家映射、游戏状态转换）；确保规则函数边界条件有测试。
10. 服务端测试补齐：对 `RoomManager`、`GameManager`、`DeckManager` 做单元测试；增加 Socket 行为集成测试（创建房间、加入、准备、开始、出牌、抽牌、挑战、结束回合、离开/断线）。
11. 客户端测试补齐：对 `GameContext` 与核心页面（Lobby/Game）做行为测试（UI 状态、按钮可用性、事件触发、错误提示）；Mock Socket 事件回放验证状态更新。
12. 端到端验收用例：以“2-4 人完整对局”为主线，补充关键异常路径（玩家断线、挑战失败/成功、连续惩罚、抽牌无可出）；形成验收清单并对照测试覆盖。
13. 文档与规范收尾：更新 README 或新增 `docs/testing.md`（如有需要）说明测试运行方式、规范与覆盖范围；记录关键架构变更与约束。

## 交付物
- 冗余清理清单与已删除项记录
- 架构调整说明（模块职责与关键移动/合并点）
- 完整测试体系（脚本、规范、单测与集成测试）
- 前后端缺失功能补齐并可通过 `npm run build`
